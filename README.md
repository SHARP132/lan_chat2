# LAN Чат

Локальный мессенджер для общения внутри одной Wi-Fi сети, без сервера.
Устройства находят друг друга автоматически (UDP-broadcast) и общаются напрямую (TCP).

## Как устроено

- **Обнаружение** — каждое устройство раз в 2 секунды рассылает в сеть широковещательный
  пакет со своим именем и портом. Остальные устройства его слышат и показывают в списке.
- **Сообщения** — когда вы открываете чат с найденным устройством, телефон/комп напрямую
  подключается к нему по TCP и обменивается сообщениями. Интернет не нужен — всё происходит
  внутри роутера.

## Способ 1 — получить готовый .apk без установки Flutter (через GitHub)

1. Зарегистрируйся на https://github.com (если ещё нет аккаунта).
2. Создай новый репозиторий (кнопка "New repository"), любое имя, можно приватный.
3. Загрузи туда все файлы и папки из этого архива — проще всего через веб-интерфейс:
   открой репозиторий → "Add file" → "Upload files" → перетащи всё содержимое
   (включая скрытую папку `.github`, её нужно загружать как есть — на GitHub она не потеряется).
4. После загрузки зайди во вкладку **Actions** репозитория — должна сама запуститься
   сборка "Build APK" (если не запустилась — нажми "Run workflow").
5. Сборка идёт 3–5 минут. Когда появится зелёная галочка — открой этот запуск, внизу
   будет файл **lan-chat-apk** — скачай и распакуй (это zip с apk внутри).
6. Перенеси `app-release.apk` на телефон любым способом (USB, Telegram, облако),
   открой файл — телефон попросит разрешить установку из неизвестных источников
   (это нормально, разрешаешь один раз) — установится как обычное приложение.

Так apk соберётся в облаке GitHub, никакой Flutter у себя на компе ставить не нужно.

## Способ 2 — собрать самому через Flutter (если хочешь дальше дописывать код)

1. Установите Flutter SDK: https://docs.flutter.dev/get-started/install
2. Установите Android Studio (нужен для Android SDK, даже если писать будете в VS Code)
3. Проверьте, что всё настроено:
   ```
   flutter doctor
   ```
4. Создайте каркас проекта:
   ```
   flutter create lan_chat
   cd lan_chat
   ```
5. Замените файл `pubspec.yaml` на тот, что лежит в этом архиве.
6. Скопируйте папку `lib/` из этого архива в папку проекта (с заменой `lib/main.dart`).
7. Откройте `android/app/src/main/AndroidManifest.xml` и добавьте внутрь тега `<manifest>`
   (до `<application>`) три строки:
   ```xml
   <uses-permission android:name="android.permission.INTERNET"/>
   <uses-permission android:name="android.permission.ACCESS_WIFI_STATE"/>
   <uses-permission android:name="android.permission.CHANGE_WIFI_MULTICAST_STATE"/>
   ```
8. Подключите телефон по USB (включите режим разработчика + отладку по USB) или возьмите
   два устройства/эмулятора в одной Wi-Fi сети.
9. Запустите:
   ```
   flutter run
   ```

## Как проверить, что работает

1. Запустите приложение на двух устройствах в одной Wi-Fi сети.
2. На каждом введите имя и нажмите «Начать поиск в сети».
3. Через пару секунд устройства должны увидеть друг друга в списке.
4. Нажмите на собеседника — откроется чат, можно писать.

## Известные ограничения (это самый первый, минимальный вариант)

- Работает только в одной сети Wi-Fi — без сервера в интернете онлайн-режим пока не работает
  (это следующий шаг).
- Некоторые сети (кафе, кампусы, гостевой Wi-Fi) блокируют широковещательные пакеты между
  устройствами («изоляция клиентов») — в домашней сети обычно всё ок.
- Один чат активен в моменте, история сообщений не сохраняется при перезапуске.
- Без шифрования сообщений.

Когда это заработает — следующий шаг — добавить сервер-релей, чтобы общаться и не находясь
в одной сети.
